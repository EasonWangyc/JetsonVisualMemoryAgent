# Interview Pitch

## 项目一句话

基于 Jetson Orin Nano 构建端侧视觉语言记忆系统，使用 ViT/CLIP 提取图像语义 embedding，结合轻量 VLM 实现自然语言检索、场景风险分析和机器人可消费的结构化 JSON 输出，并记录端侧推理延迟、显存和吞吐表现。

## 为什么做这个

机器人和智能体系统通常需要把历史视觉输入组织成可检索的长期记忆，并把模型输出约束成下游模块能直接消费的结构化结果。本项目从端侧部署角度实现一条完整链路：图片/抽帧输入、ViT/CLIP 语义索引、VLM 精读、JSON schema 校验和性能 benchmark。

## 技术亮点

- ViT/CLIP：将图像和文本映射到同一语义空间，支持自然语言检索图片。
- VLM 精读：对 top-k 图片进行场景解释、风险判断和任务建议。
- 结构化输出：把自由文本约束成机器人/Agent 可消费的 JSON。
- 端侧部署评估：在 Jetson 上记录视觉编码、检索、VLM 生成、JSON 解析和端到端延迟。
- 安全边界：LLM 输出只作为白名单 action 建议，不直接执行任意系统命令或控制动作。

## 可以回答的面试问题

- 为什么选择 ViT/CLIP 做视觉语义索引？
- ViT patch 和 CNN feature map 的区别是什么？
- CLIP 为什么能做图文检索？
- embedding 检索和 RAG 的关系是什么？
- VLM 输出 JSON 不稳定怎么办？
- 端侧 VLM 的瓶颈在哪里？
- top-1 和 top-3 精读有什么延迟/效果权衡？
- 这个项目如何往 VLA 扩展？

## 局限和下一步

- 当前不做实时摄像头输入，只处理本地图像/抽帧，适合远程 Jetson 开发。
- 当前不训练模型，只评估推理和系统接口。
- 下一步可以接 ROS2 bridge，让机器人系统通过 topic 查询视觉记忆。
- 更进一步可以接 LeRobot/OpenVLA 数据格式，把图像、语言、状态、动作串起来。
